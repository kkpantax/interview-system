# 錄取通知單自助下載 — 部署與寄信說明

> 目的：在「繳費階段」提前提供錄取通知單，學生於落地頁輸入帳號＋護照號碼即可下載自己的通知單。
> 檔案放私有 Supabase Storage，後端 Edge Function 驗證後產生短效簽名連結，檔案路徑不外露。

## 二、Supabase：建立私有 bucket（請在 Dashboard SQL Editor 執行）

```sql
-- 建立私有 bucket（不公開、不可匿名列出）
insert into storage.buckets (id, name, public)
values ('admission-letters', 'admission-letters', false)
on conflict (id) do nothing;
```

> 不需要另外開 RLS 政策：Edge Function 用的是 service_role key（繞過 RLS）。
> 保持私有即可，任何 anon key / 未授權者都讀不到檔案。

---

## 三、上傳通知單檔案到 bucket

1. Supabase Dashboard → Storage → 進入 `admission-letters` bucket。
2. 把我產出的 **`storage_upload/` 資料夾內 183 個 `帳號.pdf`** 全部拖進 bucket 根目錄。
   （檔名一定要是「帳號.pdf」，例如 `11510003.pdf`，Edge Function 靠這個對應。）
3. **7/24 那批**通知單做好後，一樣改名成 `帳號.pdf` 拖進同一個 bucket 即可，
   落地頁會自動變成可下載（不需改程式、不需改資料表）。

## 四、Vercel 環境變數

Edge Function 用的是既有的 `SUPABASE_SERVICE_ROLE_KEY`（`api/login.js`、`api/reset.js` 已在用），
**不需要新增任何環境變數**。直接 `git push` 到 main 就會自動部署。

## 五、測試方式

1. 部署後開 `https://interview-system-fawn.vercel.app/#/letter`。

> 落地頁預設語言可用連結參數帶入，讓學生一點開就是自己的語言：
> 越南 `?lang=vi`、印尼 `?lang=id`、中文 `?lang=zh`、英文 `?lang=en`（不帶則預設越南文）。
> 寄信名單 Excel 已有「語言」欄可對應。
2. 用一位「已上傳通知單」的學生：輸入其帳號 + 護照號碼 → 應直接下載 PDF。
3. 用一位「尚未上傳通知單」的學生（例如 7/24 那批）：應顯示「7/24 開放」提示，不會下載。
4. 故意輸入錯的護照號碼 → 應顯示「帳號或護照號碼不正確」。

> 注意：學生 **11510416 韋德孝（高雄）** 目前 `applications` 沒有護照號碼，
> 會無法通過驗證。請先在系統補上他的護照號碼，或單獨處理。

## 六、寄「下載通知」信給學生（129 人）

**寄送對象**：進入繳費階段、尚未繳費、且通知單已備妥的 **129 人**（名單見 `寄信名單_129.xlsx`）。
已繳費者不寄；7/24 那批（32 人）等通知單上傳後再寄。

**寄送入口（已做進系統）**：報到後台 `#/onboard-admin` →「繳費（步驟②）」分頁 → 名單上方
「✉ 寄送錄取通知單下載通知」按鈕。點開後就是你熟悉的寄信視窗：逐人雙語信（母語在前、中文在後）、
可逐列改語言、① 建立草稿到公務信箱 → 逐封檢查 → ② 送出本批。此信獨立計次（與繳費通知分開），
連結會依各生語言自動帶 `?lang=`、並帶入帳號，模板已內建（不需手動貼）。下方範本僅供參考／存查。
